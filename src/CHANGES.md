# LCBridge — изменения (1.1.0)

Все правки сделаны поверх read-only геттеров игры — в игру ничего не пишется.
Соберётся без новых зависимостей (как и раньше, против GameLibs/BepInEx).

## Новые файлы

- `src/EnemyResolver.cs` — единый резолвер имени врага. Через рефлексию (без
  жёсткой ссылки на мод) определяет турель на голове (ToilHead) и добавляет
  суффикс `+Turret` к имени: `Spring` → `Spring+Turret` (Toil-Head),
  `Manticoil` → `Manticoil+Turret` (Manti-Toil). Если ToilHead не установлен —
  работает как раньше. Используется во всех местах чтения монстров.

## Монстры

- Чтение имён монстров (`TickStats`, `GetMonsters`, `GetMonstersRaw`,
  определение «убийцы») переведено на `EnemyResolver.Resolve(ai)` — имена
  согласованы с системой иконок оверлея.
- `GameState.GetMonsterInstances()` — живые монстры с instanceID, чтобы считать
  РЕАЛЬНОЕ число уникальных особей за забег.
- `RunStats`: новое поле `count` у монстров — сколько реально заспавнилось
  (по уникальным instanceID). Список монстров в JSON теперь сортируется по
  числу особей; `sec` (время присутствия) сохранён.

## Ловушки (новое)

- `GameState.GetTraps()` — находит `Turret`, `Landmine`, `SpikeRoofTrap`
  (ваниль) через `FindObjectsOfType`, возвращает `["Turret x2","Landmine x4"]`.
- `BridgeTicker` добавляет в payload поле `"traps":[...]`.

## Лут — накопительный (собрано за игру)

- `GameState.GetShipScrapItems()` — доставленный на корабль скрап как пары
  (instanceID -> ценность).
- `RunStats` копит `_collectedScrap` по уникальным id: каждый предмет учитывается
  один раз, сумма НЕ падает после продажи компании. `money` по квотам и `profit`
  по лунам считаются от этой накопительной суммы.

## Смерти на квотах (все игроки)

- `RegisterDeath` считает смерть каждого игрока; защита от повторных вызовов
  `KillPlayer` для уже учтённого мертвеца (без фантомных +1). Общий счётчик и
  счётчик квоты согласованы.
- `GameState.OnNewRound()` вызывается на `StartGame` (старт дня) и чистит набор
  «умерших в этом раунде», чтобы повторная смерть игрока в другой день той же
  квоты считалась как новая.

## Совместимость

- Старый оверлей с новым мостом: всё работает (новые поля просто игнорируются).
- Новый оверлей со старым мостом: count -> 1 на монстра, лут по-старому, ловушек нет.
  Рекомендуется обновить обе стороны.
