# LCBridge — мост данных Lethal Company → стрим-оверлей

Свой BepInEx-мод, который отдаёт в браузерный оверлей то, чего НЕ даёт StreamOverlays:

- **Смерти** — реальный счётчик за забег (патч `PlayerControllerB.KillPlayer`)
- **Живые / всего** в экипаже (X/4)
- **HP** локального игрока
- **Полная погода WeatherTweaks** (комбо вроде "Eclipsed + Flooded") — через рефлексию
- **Ивент Brutal Company дня** — через рефлексию (см. оговорку ниже)

Данные шлются по WebSocket на `ws://localhost:8181` (порт меняется в конфиге).
Работает **вместе** со StreamOverlays — не конфликтует, у них порт 8000/8080, у нас 8181.

---

## ⚠️ Важно про рефлексию (честно)

Смерти, живые, HP, ванильная и базовая погода, луна — читаются из игры **надёжно**.

Ивент Brutal Company и комбо-погода WeatherTweaks читаются **через рефлексию** по чужим
модам, потому что у них нет публичного API. Это работает по принципу «угадай поле»:
мод ищет в их сборках типы и поля по списку вероятных имён. Если авторы этих модов
переименуют внутренние поля в новой версии — чтение ивента/комбо-погоды может отвалиться
(всё остальное продолжит работать). Если отвалится — смотри лог: строки `[reflection]`
покажут, что нашлось, а что нет, и мы подправим список имён-кандидатов.

## Сборка (на твоём ПК)

В этой среде я не могу скомпилировать мод — нет .NET SDK и игровых DLL. Собери сам, это 5 минут.

### 1. Поставь .NET SDK 8
https://dotnet.microsoft.com/download — поставь SDK (не только runtime).
Проверь: `dotnet --version` → должно показать 8.x.

### 2. Собери

```bash
cd LCBridge
dotnet build -c Release
```

`LCBridge.csproj` тянет ссылки на BepInEx и игровые сборки из NuGet
(`LethalCompany.GameLibs.Steam`). При первой сборке скачаются пакеты — нужен интернет.

Готовый файл будет в `bin/Release/LCBridge.dll`.

### Если NuGet GameLibs не подтянулся

Открой `LCBridge.csproj`, закомментируй блок `<PackageReference ... GameLibs ...>` и
`UnityEngine.Modules`, раскомментируй блок `<!-- РУЧНОЙ ВАРИАНТ ссылок -->` ниже и впиши
путь к своей установке игры (папка `Lethal Company_Data\Managed`). Затем снова `dotnet build`.

## Установка

1. В профиле r2modman найди папку:
   `...\r2modmanPlus-local\LethalCompany\profiles\<твой профиль>\BepInEx\plugins\`
2. Создай там папку `LCBridge` и положи в неё `LCBridge.dll`.
3. Запусти игру через r2modman.
4. В логах ищи: `WebSocket-сервер моста запущен на ws://localhost:8181`.

---

## Подключение оверлея

Оверлей `lethal_overlay_live.html` обновлён: он слушает ОБА источника одновременно —
StreamOverlays (8000) для луны/квоты/лута и LCBridge (8181) для смертей/погоды/ивента.
Ничего настраивать не нужно, если порты дефолтные.

Если поменял порт моста в конфиге — впиши новый в панели оверлея ([H] → «Порт моста»).

---

## Что патчится (для прозрачности)

- `PlayerControllerB.KillPlayer` (Postfix) — счётчик смертей
- `StartOfRound.ResetShip` (Postfix) — сброс счётчика при новом файле/банкротстве

Никаких изменений в геймплей мод не вносит — только читает состояние и отдаёт наружу.
